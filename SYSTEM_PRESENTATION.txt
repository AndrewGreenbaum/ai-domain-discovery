WHAT IT DOES:
-------------
This system automatically discovers, validates, and scores NEW AI startups within
hours of them registering their .ai domain - before they appear on any startup
database, news site, or investor radar.

VALUE PROPOSITION:
------------------
- First-mover advantage: Find startups 30-90 days before Crunchbase/PitchBook
- Zero manual research: Fully automated 24/7 pipeline
- AI-powered analysis: Claude 3.5 Sonnet evaluates each domain
- Investment signals: Identifies high-quality launches vs parking pages

-----------------------------------
- Total domains tracked: 84
- Live websites found: 69 (82.1% hit rate)
- LLM evaluations completed: 15
- Average processing: ~50 domains/day capacity
- Uptime: 99.2% (EC2 t3.medium)

2.3 DISCOVERY SOURCES (ALL FREE)
--------------------------------

Source #1: Certificate Transparency Logs
- Primary: crt.sh API
- Secondary: censys.io, Google CT, Cloudflare CT
- Method: Query for new .ai SSL certificates
- Latency: 2-6 hours from SSL issuance
- Volume: 60-70% of discoveries

Source #2: GitHub API
- Method: Search repos/READMEs for .ai domains
- Rate limit: 5,000 requests/hour (with token)
- Volume: 10-20% of discoveries

Source #3: Startup Directories
- Product Hunt RSS feed
- YCombinator directory scraping
- Volume: 10-15% of discoveries

Source #4: DNS Pattern Enumeration
- Method: Check common AI naming patterns
- Examples: {tech-word}.ai, {verb}.ai
- Volume: 5-10% of discoveries

LLM Decision Points:

  3. What's the quality score (0-100)?
  4. Red flags detected?
  5. Category classification (AI/ML, SaaS, etc.)

SECTION 5: CURRENT LIMITATIONS & BOTTLENECKS
================================================================================

5.1 COMPUTE BOTTLENECK
----------------------

Current: AWS EC2 t3.medium
  - vCPUs: 2
  - RAM: 4GB
  - Burst credits: Limited

Impact:
  - Can only run 3 concurrent validations
  - Screenshot capture: 1 at a time (memory constraint)
  - Large discovery runs timeout after 10 minutes

Solution: Upgrade to t3.large or m5.large


5.2 LLM RATE LIMITS
-------------------

Current: Claude API (standard tier)
  - Requests/minute: 60
  - Tokens/minute: 100,000

Impact:
  - Can only evaluate ~50 domains/hour with LLM
  - Vision analysis is expensive ($0.01-0.02/domain)
  - Burst discovery days hit rate limits

Solution: Higher API tier or batch processing


5.3 DATA SOURCE LIMITATIONS
---------------------------

CT Logs:
  - crt.sh rate limit: 100 requests/minute
  - Censys.io: Requires paid account for bulk
  - Google CT: Complex authentication

GitHub:
  - Without token: 60 requests/hour
  - With token: 5,000 requests/hour (need GitHub auth)

Startup Directories:
  - Scraping is slow and fragile
  - Product Hunt rate limits apply


5.4 STORAGE LIMITATIONS
-----------------------

Current: S3 Standard
  - Screenshots: ~500KB each
  - 100 screenshots/day = 50MB/day = 1.5GB/month

Impact:
  - S3 costs scale linearly
  - No image compression implemented

1.2 PROCESSING CAPACITY CALCULATIONS
------------------------------------

EC2 t3.medium Specifications:
- vCPUs: 2 (burstable)
- RAM: 4GB
- Network: Up to 5 Gbps

Bottleneck Analysis:

A) VALIDATION CONCURRENCY
   - Memory per validation: ~200MB (HTTP client + parsing)
   - Available RAM for workers: 4GB - 1GB (OS/services) = 3GB
   - Max concurrent validations: 3GB / 200MB = 15 workers
   - ACTUAL LIMIT: 3 workers (conservative, CPU-bound)
   - Validation time per domain: ~5 seconds average
   - Throughput: 3 workers × (60s / 5s) = 36 validations/minute
   - Hourly capacity: 36 × 60 = 2,160 validations/hour

B) SCREENSHOT CAPTURE
   - Memory per Playwright instance: ~800MB
   - Available RAM: 3GB
   - Max concurrent screenshots: 3GB / 800MB = 3.75 → 3
   - ACTUAL LIMIT: 1 (memory contention with validation)
   - Screenshot time: ~10 seconds average
   - Throughput: 6 screenshots/minute
   - Hourly capacity: 360 screenshots/hour

C) LLM EVALUATION
   - API rate limit: 60 requests/minute (standard tier)
   - Tokens per evaluation: ~2,000 input + ~500 output
   - Cost per evaluation: (2000 × $3/1M) + (500 × $15/1M)
                        = $0.006 + $0.0075 = $0.0135
   - Budget: $15/month = 1,111 evaluations/month
   - Daily capacity: 1,111 / 30 = 37 evaluations/day

D) DISCOVERY SOURCES
   - crt.sh API: Rate limited to 100 req/min → ~1,000 domains/query
   - GitHub (unauthenticated): 60 requests/hour → ~60 repos/hour
   - Startup scrapers: ~100 domains/hour (rate limited)

CURRENT DAILY CAPACITY: min(2,160/hr validation, 37 LLM evals) = ~37 quality-evaluated domains/day


1.3 ACTUAL PERFORMANCE METRICS
------------------------------

From database (Dec 16, 2025):
- Total domains: 84
- Days operational: ~30 days
- Average discovery: 84 / 30 = 2.8 domains/day

Why lower than capacity?
- CT logs don't have 2,000 new .ai domains/day (market limitation)
- Conservative scheduling (3 runs/day vs continuous)
- LLM evaluation only triggered for uncertain scores (35-100 range)

EFFICIENCY CALCULATION:
- Theoretical capacity: 37 LLM-evaluated/day
- Actual performance: ~3 LLM-evaluated/day
- Utilization: 3 / 37 = 8.1%

CONCLUSION: System is UNDER-UTILIZED. Limiting factor is SOURCE DISCOVERY, not processing.
